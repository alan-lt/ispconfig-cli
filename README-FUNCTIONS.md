## setDirectiveSnippet

```php
setDirectiveSnippet($domain_id, $snippet_id = null, $snippet_name = null)
```

Assigns a directive snippet (nginx/apache/php template) to a web domain. directive_snippets_id is a UI-plugin field that the SOAP API silently ignores (on both add and update), so this uses ISPConfig's own interface library (datalogUpdate) to update the column and queue the vhost rebuild - exactly what the panel does. A raw SQL UPDATE would change the value but not rebuild the vhost. Relies on the interface library, which soap_env.php bootstraps for every script. Pass either $snippet_id or $snippet_name. Pass $snippet_id = 0 to clear the assignment (domain uses no directive snippet).

| Param | Type | Description |
|-------|------|-------------|
| `$domain_id` | int | Web domain ID |
| `$snippet_id` | int\|null | Directive snippet ID (0 clears the assignment) |
| `$snippet_name` | string\|null | Directive snippet name |

**Returns:** `array` — Result array with 'success' plus details, or 'error' on failure
