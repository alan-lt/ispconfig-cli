## emitEvent

```php
emitEvent($event)
```

Emit one NDJSON event line to stdout and flush it immediately. The CLI output contract: every line printed to stdout is exactly one JSON object carrying a "type" discriminator ("progress", "notice", "result"). Consumers read line by line, json_decode each line and switch on type; anything that fails to decode should be ignored (stray PHP warning, etc.). flush() guarantees progress reaches the reader in real time.

| Param | Type | Description |
|-------|------|-------------|
| `$event` | array | Event payload; "type" is forced to the front |

## waitForEmptyJobQueue

```php
waitForEmptyJobQueue($server_id = 0, $sleep_seconds = 2, $timeout_seconds = 300, $stable_iterations = 3, $connection_wait = 10)
```

Wait for the ISPConfig job queue to drain (empty and stay empty). Polls monitor_jobqueue_count until it reports zero for $stable_iterations consecutive checks. A briefly unavailable API (SoapFault) is not fatal: it emits a "notice" and retries after $connection_wait seconds. Emits "progress"/"notice" NDJSON events via emitEvent() while waiting; the caller is responsible for emitting the terminal "result" from the returned array (see emitResult()).

| Param | Type | Description |
|-------|------|-------------|
| `$server_id` | int | Server ID (0 for all servers) |
| `$sleep_seconds` | int | Seconds between checks (default: 2) |
| `$timeout_seconds` | int | Maximum wait time (default: 300) |
| `$stable_iterations` | int | Consecutive zero counts required (default: 3) |
| `$connection_wait` | int | Seconds to wait after a connection error (default: 10) |

**Returns:** `array` — Result with success status

## setDirectiveSnippet

```php
setDirectiveSnippet($domain_id, $snippet_id = null, $snippet_name = null)
```

Assigns a directive snippet (nginx/apache/php template) to a web domain. directive_snippets_id is a UI-plugin field that the SOAP API silently ignores (on both add and update), so this uses ISPConfig's own interface library (datalogUpdate) to update the column and queue the vhost rebuild - exactly what the panel does. A raw SQL UPDATE would change the value but not rebuild the vhost. Relies on the interface library, which soap_env.php bootstraps for every script. Pass either $snippet_id or $snippet_name. Pass $snippet_id = 0 to clear the assignment (domain uses no directive snippet).

| Param | Type | Description |
|-------|------|-------------|
| `$domain_id` | int | Web domain ID |
| `$snippet_id` | int\|null | Directive snippet ID (0 clears the assignment) |
| `$snippet_name` | string\|null | Directive snippet name |

**Returns:** `array` — Result array with 'success' plus details, or 'error' on failure
